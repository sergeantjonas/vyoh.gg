# Docs

Three kinds of documentation for vyoh.gg:

- **[working-notes/](working-notes/)** — internal planning. Active arcs, deferred items, history, references. Start with [working-notes/open-work.md](working-notes/open-work.md) to see what's next, or [working-notes/README.md](working-notes/README.md) for the folder map.
- **[case-studies/](case-studies/)** — public write-ups. One per shipped arc worth a portfolio story; the freelance-positioning narrative.
- **[repo-conventions.md](repo-conventions.md)** — portable rules (architecture, environment, workflow).

Working notes are where ideas live before they ship; case studies are where they go public after they ship. Repo conventions are what survives beyond any single arc.
